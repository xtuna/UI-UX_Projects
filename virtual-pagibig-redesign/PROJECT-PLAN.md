# Project Plan — Master Checklist

Follow this top to bottom. Each phase links to its detailed doc. Check items off as you go — GitHub renders checkboxes, so you can literally tick these in the file and commit as proof of progress.

---

## Phase 0 — Setup
- [ ] Create a free [Figma](https://figma.com) account
- [ ] Create a free [FigJam](https://figma.com/figjam) file (comes with your Figma account) for research/synthesis
- [ ] Create this GitHub repo (public, so it's linkable in your portfolio)
- [ ] Clone it locally or edit directly on GitHub.com
- [ ] Collect your "before" screenshots of the current site/app into `assets/screenshots-before/`

## Phase 1 — Discover (Research)
📄 Full detail: [`docs/01-research.md`](./docs/01-research.md)
- [ ] Complete the heuristic evaluation table (screenshot-based)
- [ ] Summarize secondary research (Play Store reviews, legal commentary, ARTA data)
- [ ] Conduct or synthesize 3–5 user interviews
- [ ] Build an affinity map in FigJam, screenshot it into `assets/`

## Phase 2 — Define
📄 Full detail: [`docs/02-define.md`](./docs/02-define.md)
- [ ] Write 2 personas
- [ ] Build empathy maps for each
- [ ] Build a current-state journey map
- [ ] Write 4–5 "How Might We" statements

## Phase 3 — Ideate
📄 Full detail: [`docs/03-ideate-storyboard.md`](./docs/03-ideate-storyboard.md)
- [ ] Draft the shared sitemap (web + app)
- [ ] Prioritize features (MoSCoW)
- [ ] Draw a 6-panel UX storyboard for your primary persona
- [ ] Save storyboard panels into `assets/storyboard/`

## Phase 4 — Design System
📄 Full detail: [`docs/04-design-system.md`](./docs/04-design-system.md)
- [ ] Define color palette (with hex codes) in Figma styles
- [ ] Define typography scale in Figma styles
- [ ] Build core components (buttons, inputs, cards, nav, masked-data field, status badges)

## Phase 5 — Wireframes → Mockups → Prototype
📄 Full detail: [`docs/05-wireframes-mockups-prototype.md`](./docs/05-wireframes-mockups-prototype.md)
- [ ] Low-fi wireframes — App (6 screens)
- [ ] Low-fi wireframes — Web (6 screens)
- [ ] Export wireframes to `assets/wireframes/`
- [ ] Hi-fi mockups — App (using design system)
- [ ] Hi-fi mockups — Web (using design system)
- [ ] Export mockups to `assets/mockups/`
- [ ] Wire up interactive prototype — App
- [ ] Wire up interactive prototype — Web
- [ ] Get shareable Figma prototype links, paste into README

## Phase 6 — Usability Testing
📄 Full detail: [`docs/06-usability-testing.md`](./docs/06-usability-testing.md)
- [ ] Test wireframes with 5 people before moving to hi-fi (or test hi-fi if time-constrained)
- [ ] Document findings
- [ ] Note any iterations made based on findings

## Phase 7 — Case Study Write-Up
📄 Full detail: [`docs/07-case-study-template.md`](./docs/07-case-study-template.md)
- [ ] Write the case study following the template
- [ ] Publish on Behance / Notion / personal site, linking back to this GitHub repo
- [ ] Update this README's "Links" section with final URLs
