# 07 — Case Study Write-Up Template

Use this structure for your final portfolio piece (Behance, Notion, personal site, or an expanded version of this repo's README). Recruiters skim — front-load the outcome.

---

### 1. Header
Title, your role, tools used, duration, platforms (Web + Mobile App). Link this GitHub repo.

### 2. Overview
2–3 sentences: what Virtual Pag-IBIG is, what you redesigned, why both platforms.

### 3. The Problem
Lead with your heuristic evaluation table — real screenshot evidence is a strong, credible opener. Add the ARTA ranking and app store review context for weight.

### 4. Research
Show personas and the journey map (one slide each, not walls of text). Link to `docs/01-research.md` and `docs/02-define.md` for full detail.

### 5. The Storyboard
Include your 6-panel storyboard — this is a strong visual/narrative moment in the case study, shows empathy beyond just screens.

### 6. Design Process
Wireframes → usability test findings → what changed → hi-fi screens, side-by-side before/after. Include a table:

| Heuristic Issue | Screen | Resolution in Redesign |
|---|---|---|
| #1 Blank header | All app screens | Added Page Header component with title |
| #2 Exposed MID | Account, Savings | Masked Data Field component |
| ... | | |

### 7. Design System
Show your color palette, typography, and 2-3 key components — demonstrates systems thinking, not just individual screens.

### 8. Prototype
Embed or link both clickable Figma prototypes (app + web).

### 9. New Feature Spotlight
Explain the "Report a Posting Issue" flow — why you added it, tied to the Respicio legal commentary on posting errors. This shows you can identify and design for needs beyond just fixing what's visibly broken.

### 10. Outcome / Impact
Be honest — no invented metrics. Frame it as: *"This redesign directly resolves 10 of 12 heuristic violations identified in the original product audit, and introduces one new flow addressing a documented, unresolved member pain point."*

### 11. Reflection
2–3 sentences: what you'd test next (e.g., A/B test the masked-MID reveal pattern, validate the new posting-issue flow with real affected users).
