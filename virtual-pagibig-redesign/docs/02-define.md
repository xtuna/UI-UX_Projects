## 2.3 Current-State Journey Map

Example: checking loan status.

| Stage | Open app/site | Navigate | View Status | Understand Status | Next Step |
|---|---|---|---|---|---|
| Action | Logs in | Taps product grid or sidebar | Finds "View Loan Status" | Reads result | Decides what to do |
| Pain point | Alarmed by failed-login message | Grid doesn't distinguish "view" vs "apply" | Blank header gives no confirmation of correct screen | Status language may be unclear | No clear next-step CTA |
| Emotion | 😰 anxious | 😕 unsure | 😐 uncertain | 😕 unsure | 😤 frustrated |

Build this as a horizontal swimlane diagram in FigJam (stage columns, emotion curve underneath). Export to `assets/`.

## 2.4 How Might We Statements

- How might we replace alarming security messaging with clear, reassuring system status?
- How might we make read-only vs. editable data visually unambiguous?
- How might we group features by user goal (view / apply / manage / get help) instead of by product silo?
- How might we make every screen self-identifying, so users always know where they are?
- How might we give users a real actionable path when a contribution or posting issue occurs?
